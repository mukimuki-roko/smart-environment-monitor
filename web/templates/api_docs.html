API REFERENCE

使い方

JSON API は Content-Type: application/json を返します。センサーデータは日時の新しい順に返ります。

GET

/api/sensor-data

保存済みの全センサーデータを取得します。

応答は csv_path、fieldnames、field_labels、rows、row_count を含みます。

curl 例を表示
curl {{ request.host_url.rstrip('/') }}/api/sensor-data
/api/sensor-data

実行結果

{}
GET

/api/sensor-data/search

センサーデータを検索します。条件を省略すると全件を返します。

パラメータ内容
client_id / region検索文字列。既定は部分一致。
client_id_match / region_matchcontains（既定）または equals。
datetime_from / datetime_toISO 8601 の開始／終了日時。
temperature / humidity / pressure / co2完全一致する数値。
*_min / *_max各測定値の最小／最大値。

成功時は全データ API の項目に加えて、適用した filters を返します。不正な条件は 400 {"error":"..."} を返します。

curl 例を表示
curl "{{ request.host_url.rstrip('/') }}/api/sensor-data/search?client_id=TK&temperature_min=25"
基本条件
数値条件を指定する
/api/sensor-data/search

実行結果

{}
GET

/api/health

端末ごとの最新ヘルス状態を取得します。

client_id と region を指定すると、各項目の部分一致で絞り込みます。応答は offline_after_seconds、filters、clients を含みます。

curl 例を表示
curl {{ request.host_url.rstrip('/') }}/api/health
検索条件
/api/health

実行結果

{}
GET

/api/health/<client_id>/download

指定端末の保存済みヘルス履歴をCSVでダウンロードします。

端末IDは完全一致で検索します。CSVは受信時刻順の全履歴を既存の保存CSVと同じ列順で返し、UTF-8 BOMを含みます。該当履歴がない場合は 404 {"error":"health history not found"} を返します。

curl 例を表示
curl -OJ {{ request.host_url.rstrip('/') }}/api/health/tk240131/download
/api/health/tk240131/download
SSE

/api/health/stream

ヘルス更新を Server-Sent Events で通知します。最初に retry: 3000 を返し、更新時は event: health、data: updated を送信します。

curl 例を表示
curl -N {{ request.host_url.rstrip('/') }}/api/health/stream
/api/health/stream
